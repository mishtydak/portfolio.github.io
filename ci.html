Course Project Introduction


This project addresses challenges in travel management by focusing on user-centric solutions. It leverages efficient algorithms for preference handling, recommendation systems, and sustainable travel. The findings have helped shape innovative business cases for better user experiences.

Problem Definition and Team Details

The project focuses on solving key issues in travel planning, such as preference sorting, personalized recommendations, sustainable transport options, and emergency services. The team roles and contributions are outlined below:

  • MISHTY - Responsible for listing user preferences and implementing efficient sorting mechanisms using Array and Dijkstra's algorithm.
  • BHAGYASHRI - Develops ranking-based sorting for attractions and manages transport scheduling using Bubble Sort and Insertion Sort.
  • DIYA - Works on carpooling solutions and shop categorization using Disjoint Set Union and Switch Cases.
  • ANANYA - Handles tourist package sorting and transport mode selection using Selection Sort and Switch Cases.

Business Cases

  • Listing Preferences: Enables users to add and update their preferences for destinations or activities (MISHTY - Array and Dijkstra's algorithm).
  • Tourist Packages: Allows sorting of travel packages based on price (ANANYA - Selection Sort).
  • Tourist Attractions: Displays attractions sorted by rank or popularity (BHAGYASHRI - Bubble Sort).
  • Transportation Modes: Helps users select suitable modes for inter-city travel (ANANYA - Switch Case).
  • Carpooling Options: Matches users with potential carpool partners for shared travel (DIYA - Disjoint Set Union).
  • Transport Timings: Sorts transport schedules for easier trip planning (BHAGYASHRI - Insertion Sort).
  • Shop Local: Promotes local shopping with category-based sorting (DIYA - Switch Case).
  • Emergency Services: Provides quick access to emergency contacts like hospitals and police (MISHTY - Switch Case).
